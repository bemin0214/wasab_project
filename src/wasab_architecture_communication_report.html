WaSaB SW Architecture 통신 리포트

SW_Architecture 다이어그램의 3단 구조(UI, Service, Device)를 기준으로 각 컴포넌트가 어떤 데이터를 주고받는지 임의 샘플 값으로 정리한 HTML 리포트입니다.

1. 아키텍처 단 구성

UI

PC AdminGUI
  • 로봇 카메라 프리뷰 표시
  • Pick, Place, Detect, Capture, Stop 명령 입력
  • 서비스 상태와 추론 결과 확인
Mobile UserGUI
  • 사용자용 명령/상태 화면
  • 작업 요청 또는 간단한 제어 이벤트 생성

Service

WasabServer / WasabWebService
  • HTTP API와 AdminGUI 제공
  • 카메라 스트림 수신 및 최신 프레임 보관
  • YOLO 검출, 명령 큐, 결과 저장
WasabOpService
  • YOLO bbox 중심을 3D 목표점으로 변환
  • Hand-Eye, intrinsic, TCP offset 적용
  • Flange command 생성
WasabAIServer
  • AI 서비스 확장 영역
  • face_db, ai_service 데이터 사용
WasabDB
  • 운영 로그, 사용자/상태 데이터 저장 영역
  • 현재 리포트는 임의 예시 데이터 사용

Device

WasabBot / WasabArm
  • 카메라 프레임 캡처
  • MyCobot pose, gripper, motion 제어
  • 서버에서 받은 Flange command 안전 검증
WasabBot / WasabLeg
  • 이동 베이스, 센서, navigation, lamp/LED
  • ROS2 패키지 기반 하위 장치 제어
WasabBot / WasabHead
  • 헤드/표정/상호작용 장치 영역
  • AI 또는 UI 이벤트와 연동 가능

컴포넌트 이름은 현재 ~/dev_ws/wasab-manipulation 폴더 구조와 실행 코드 기준으로 정리했습니다.

2. 전체 데이터 흐름

UI AdminGUI / UserGUI 사용자가 Pick, Detect, Place 같은 명령을 누르고, 카메라 스트림과 결과를 봅니다.
→
Service WasabWebService 명령을 큐에 넣고, WasabArm이 업로드한 최신 프레임을 보관하며, YOLO와 OPService를 호출합니다.
↔
Device WasabArm / WasabLeg / WasabHead 실제 센서와 액추에이터를 제어합니다. Arm은 카메라 이미지와 pose를 서버로 보내고 명령을 받아 실행합니다.

3. 컴포넌트 간 통신 매트릭스

From To 방식 데이터 예시 목적
PC AdminGUI WasabWebService HTTP GET /camera-view client=operator-pc, session_id=admin-001 제어 화면과 실시간 프리뷰 UI 표시
PC AdminGUI WasabWebService HTTP POST /wasab-arm/commands {"command":"pick","source":"admin_gui"} Arm이 소비할 원격 명령 등록
WasabArm WasabWebService HTTP stream command stream {"id":42,"command":"pick","pending":0} 브라우저에서 누른 명령을 Arm client가 지속 수신
WasabArm WasabWebService UDP 8001 frame_id=38120, chunk=4/16, jpeg_bytes=21760 카메라 프리뷰 프레임을 낮은 지연으로 업로드
WasabArm WasabWebService HTTP POST /camera-frame image=frame.jpg, width=640, height=480 UDP 실패 시 프리뷰 fallback 업로드
WasabArm WasabWebService HTTP POST /v1/grasp-plan image=frame.jpg + robot_state.flange_coords Pick을 위한 YOLO 검출과 3D 파지 계획 요청
WasabWebService WasabOpService local call bbox=[218,146,356,301], flange=[147.2,52.7,242.2,-177.51,5.1,-94.2] 2D 검출 중심을 Base 좌표계 목표점으로 변환
WasabOpService WasabWebService return flange_command=[152.6,61.8,160.0,-177.51,5.1,-94.2] 로봇팔이 실행할 목표 pose 반환
WasabWebService WasabDB file/db write result.json, annotated.jpg, command_log 운영 로그와 추론 결과 저장
WasabAIServer WasabDB / face_db local/db face_id=guest-07, confidence=0.91 사용자 인식 또는 AI 기능 확장 데이터 조회
WasabArm MyCobot / Gripper serial send_coords([152.6,61.8,160.0,-177.51,5.1,-94.2], speed=35) 안전 검증 후 실제 로봇 동작 수행
WasabLeg ROS2 navigation / sensor packages ROS2 topic/service /cmd_vel={linear_x:0.12, angular_z:0.03} 이동, 센서, 램프/LED 등 하위 장치 제어

5. 프리뷰 / Detect / Capture 데이터 흐름

작업 흐름 샘플 데이터 결과
실시간 프리뷰 WasabArm → WasabWebService → AdminGUI UDP frame_id=38121, jpeg=21KB, fps=4.0 /camera-frame/stream.mjpg로 브라우저 표시
Detect 버튼 AdminGUI → WasabWebService → YOLO conf=0.5, imgsz=640, label=box 프레임 위에 bbox와 confidence overlay
Capture 버튼 AdminGUI → WasabWebService filesystem filename=jetcobot_capture_20260710_175520_240102.jpg runtime/app/components/capture/에 저장
로그 저장 WasabWebService → laptop_detect_logs raw.jpg, annotated.jpg, result.json save_results=true일 때 요청별 저장

6. 파일/설정 기준 위치

단 컴포넌트 현재 경로 주요 파일
Device WasabArm Device/WasabBot/WasabArm run_client.py, robot_client/main.py, config/client_config.ini
Service WasabWebService runtime Service/WasabServer/runtime run_server.py, app/main.py, config/server_config.ini
Service YOLO model Service/WasabServer/runtime/models best.pt
Service Calibration Service/WasabServer/runtime/calibration camera_intrinsic_charuco.npz, auto_handeye_result_20260619_162107.json
UI PC AdminGUI UI/pc/admin_gui 및 runtime /camera-view 브라우저 기반 제어 화면

7. 정상 동작 판단 기준

확인 지점 정상 샘플 실패 시 확인
Service 시작 GET /health → {"status":"ok","runtime":"laptop-local"} 포트 8000 충돌, models/best.pt, calibration 경로
Device 연결 python3 check_server_connection.py → [OK] runtime=laptop-local grasp_server_url IP, 방화벽, 같은 네트워크 여부
프리뷰 /camera-view에서 640x480 영상 표시 카메라 ID, UDP 8001, HTTP fallback, 프레임 크기
Pick 계획 plan.flange_command가 6개 숫자로 반환 YOLO 미검출, bbox 이상, Hand-Eye 오차, 안전 영역 초과
실제 Arm 동작 safety_check=pass 후 send_coords 실행 dry_run, serial port, gripper 값, workspace limit

이 리포트의 숫자와 request id는 설명용 임의 값입니다. 실제 테스트 결과는 runtime/laptop_detect_logs와 AdminGUI 화면에서 확인합니다.
